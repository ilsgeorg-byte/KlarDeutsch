# 🛡️ Технические мандаты KlarDeutsch

Этот файл содержит фундаментальные правила разработки и обработки данных в проекте. Gemini CLI обязан следовать им в каждой сессии.

### 1. 🇩🇪 Лингвистическая корректность (Немецкий язык)
- **Глаголы (Verben)**:
  - НИКОГДА не добавлять артикль (`article`). Поле должно быть `""`.
  - НИКОГДА не добавлять форму множественного числа (`plural`). Поле должно быть `""`.
  - ОБЯЗАТЕЛЬНО заполнять `verb_forms` (4 формы через запятую).
- **Существительные (Substantive)**:
  - ОБЯЗАТЕЛЬНО определять артикль (`der`, `die`, `das`).
  - ОБЯЗАТЕЛЬНО добавлять форму множественного числа (`plural`), если она существует. Если нет — писать `-`.
  - Поле `verb_forms` должно быть ПУСТЫМ (`""`).
- **Прилагательные (Adjektive)**:
  - Артикль и множественное число должны быть ПУСТЫМИ (`""`).

### 2. 🤖 Работа с ИИ (Enrichment)
- **Примеры (Examples)**: Всегда запрашивать у ИИ минимум 3 примера. Сохранять их в поле `examples` (формат JSONB).
- **Перевод (RU)**: Указывать несколько значений через запятую, если слово многозначно.
- **Темы (Topics)**: Использовать ИИ для классификации слов по темам (Еда, Работа, Путешествия и т.д.).

### 3. 💾 База данных (PostgreSQL)
- Перед массовыми обновлениями через ИИ всегда проверять консистентность полей (например, чтобы `verb_forms` не попадали в существительные).
- Использовать `ai_checked_at` для отслеживания прогресса проверки.

### 4. 📂 Архитектура
- **Flask (api/)**: Отвечает за основную логику, одиночное обогащение и работу со словами.
- **Next.js (app/api/)**: Отвечает за админские инструменты (статистика, массовая проверка).
- Схемы Pydantic в `api/schemas.py` являются эталонными для обмена данными.
