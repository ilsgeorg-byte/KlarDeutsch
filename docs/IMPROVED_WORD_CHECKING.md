# 🚀 Улучшенная проверка слов в KlarDeutsch

## 📋 Обзор

Этот набор инструментов улучшает проверку слов в базе данных KlarDeutsch, решая следующие проблемы:

- ✅ **Множественное число существительных** - проверка и исправление форм
- ✅ **Формы глаголов** - проверка Infinitiv, Präteritum, Partizip II
- ✅ **Точность переводов** - улучшение качества русских переводов
- ✅ **Комплексная проверка** - улучшенная версия основного скрипта

## 🛠️ Скрипты

### 1. `improve_word_checking.py` - Улучшенная комплексная проверка
Проверяет:
- Тип слова (существительное, глагол, прилагательное)
- Артикли
- Формы глаголов
- Формы множественного числа
- Точность перевода

**Запуск:**
```bash
cd api
python improve_word_checking.py
```

### 2. `check_noun_plural.py` - Проверка форм множественного числа
Специализированный скрипт для проверки и исправления форм множественного числа существительных.

**Запуск:**
```bash
cd api
python check_noun_plural.py
```

### 3. `check_verb_forms.py` - Проверка форм глаголов
Специализированный скрипт для проверки и исправления форм глаголов (Infinitiv, Präteritum, Partizip II).

**Запуск:**
```bash
cd api
python check_verb_forms.py
```

### 4. `check_translation_accuracy.py` - Проверка точности переводов
Специализированный скрипт для оценки и улучшения точности русских переводов.

**Запуск:**
```bash
cd api
python check_translation_accuracy.py
```

## ⚙️ Настройка

### 1. Установите зависимости
```bash
pip install openai psycopg2-binary python-dotenv
```

### 2. Добавьте API ключ в `.env.local`
```env
GROQ_API_KEY=ваш_groq_api_ключ
POSTGRES_URL=ваш_postgres_url
```

### 3. Получите Groq API ключ
1. Зайдите на https://console.groq.com/keys
2. Создайте аккаунт
3. Создайте новый API Key
4. Добавьте в `.env.local`

## 🚀 Использование

### Режим проверки (без изменений в БД)
Все скрипты по умолчанию работают в режиме проверки (`DRY_RUN = True`). Они анализируют слова и показывают ошибки, но не изменяют базу данных.

### Режим исправления (с изменениями в БД)
Чтобы применить изменения к базе данных:

1. Откройте нужный скрипт
2. Измените `DRY_RUN = False`
3. Запустите скрипт

**Пример:**
```python
DRY_RUN = False  # Было: True
```

## 💡 Рекомендации

1. **Запускайте по очереди** - начните с комплексной проверки, затем специализированные
2. **Создайте бэкап БД** перед массовыми изменениями
3. **Проверяйте результаты** - ИИ может иногда ошибаться
4. **Запускайте ночью** - проверка большого количества слов занимает время
5. **Используйте DRY_RUN** сначала для анализа, потом для исправлений

## 🔄 Порядок запуска для максимального эффекта

1. `check_translation_accuracy.py` - улучшить переводы
2. `check_noun_plural.py` - добавить/исправить формы множественного числа
3. `check_verb_forms.py` - исправить формы глаголов
4. `improve_word_checking.py` - финальная комплексная проверка

## 🔄 Сброс статуса проверки

Если вы хотите запустить проверку заново с начала (для всех слов), используйте скрипт сброса:

```bash
cd api
python reset_ai_check_status.py
```

Этот скрипт сбросит статус проверки для всех слов в базе данных, после чего проверка будет начинаться с начала.

## 🔍 Поиск проблемных слов

Для поиска существительных без форм множественного числа используйте:

```bash
cd api
python find_missing_plurals.py
```

Этот скрипт покажет список существительных, у которых отсутствует форма множественного числа.

## 🛠️ Исправление ошибок

Для исправления известных ошибок в словах (например, когда у существительного неправильно указаны формы глагола) используйте:

```bash
cd api
python fix_word_errors.py
```

Этот скрипт исправляет распространенные ошибки, такие как:
- Неправильные формы глагола у существительных
- Неправильные артикли у глаголов
- Пример: "Guten Morgen" с формой "kritisieren, kritisierte, kritisiert"

## 🚨 Ошибки

### `GROQ_API_KEY не найден`
Добавьте в `.env.local`:
```env
GROQ_API_KEY=ваш_ключ_здесь
```

### `Rate limit exceeded`
Подождите минуту и повторите попытку - Groq имеет ограничения на количество запросов.

### `POSTGRES_URL не найдена`
Проверьте `.env.local`:
```env
POSTGRES_URL=postgresql://...
```
