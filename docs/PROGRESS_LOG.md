## 🔧 Исправление форм слов: множественное число и глаголы (3 апреля 2026, вторая половина дня)

### Проблема
Пользователь прошёл все слова через "AI проверка слов" в админке и обнаружил:
- **Существительные без множественного числа**: например, `die Unterschrift` (Подпись) - нет формы Plural
- **Глаголы без форм**: отсутствуют формы глаголов (Infinitiv, Präteritum, Partizip II)
- **Прилагательные определялись как глаголы**: например, `neu` (новый), `groß` (большой) - у них не должно быть verb_forms

### Решение

#### 5. AI проверка: Улучшенный промпт для определения типа слова
   - **Обновлён `ENRICH_PROMPT`** в `app/api/admin/check-words/route.ts`:
     - Явное указание AI определять тип слова (noun/verb/adjective/phrase)
     - Для **существительных**: ОБЯЗАТЕЛЬНО заполнять `corrected_article` и `corrected_plural` (или `—` если нет мн.ч.)
     - Для **глаголов**: ОБЯЗАТЕЛЬНО заполнять `corrected_verb_forms`
     - Для **прилагательных**: все поля пустые (пропускаются)
     - Добавлены примеры для каждого типа слова

#### 6. Логика обработки в AI check endpoint
   - **Обновлён POST обработчик** в `app/api/admin/check-words/route.ts`:
     - Для существительных: заполняет `article` и `plural` (если отсутствуют или изменились)
     - Для глаголов: заполняет `verb_forms` (если отсутствуют или изменились)
     - Для прилагательных: пропускает (не добавляет verb_forms)
     - Маркер `—` для слов без множественного числа записывается в БД

#### 7. Фронтенд: Отображение маркера отсутствия мн.ч.
   - **Обновлены компоненты**:
     - `app/components/WordCard.tsx`: если `plural === '—'`, показывает "Pl: **нет мн.ч.**" (серым цветом)
     - `app/dictionary/[id]/page.tsx`: аналогичное обновление для страницы словаря
     - `app/admin/words/page.tsx`: обновлён placeholder в форме: `"напр. die Häuser или — если нет мн.ч."`

#### 8. Скрипты для массового заполнения форм

##### `tools/check_missing_forms.py` - Диагностика
- Показывает статистику: сколько слов без `plural` и `verb_forms`
- Выводит примеры таких слов (первые 15)
- Обновлён для работы с AI-определением типа слов

##### `tools/smart_fill_missing_forms.py` - Умное заполнение через AI
- **AI сам определяет тип слова** (существительное/глагол/прилагательное/фраза)
- Заполняет соответствующие поля:
  - Существительные: `article` + `plural` (или `—` если нет мн.ч.)
  - Глаголы: `verb_forms` в формате "Infinitiv, Präteritum, Partizip II"
  - Прилагательные: пропускает (им не нужны эти формы)
  - Фразы: пропускает
- **Использует модель `qwen-3-32b`** (быстрее и стабильнее, чем `llama-3.3-70b-versatile`)
- **Автоматическая обработка лимитов API**:
  - До 3 повторных попыток при rate limit
  - Экспоненциальная задержка (2с, 4с, 6с...)
  - Обработка сетевых ошибок и таймаутов
- **Прогресс и статистика**:
  - Показывает прогресс каждые 10 слов
  - Время выполнения и скорость (слов/сек)
  - Количество ошибок и повторных попыток
- **Пакетная обработка**: BATCH_SIZE = 100 (оптимально), можно изменить

##### `tools/fill_missing_forms.py` - Старая версия
- Оставлена для совместимости
- Использует эвристики для определения типа слова
- Менее точная, чем `smart_fill_missing_forms.py`
