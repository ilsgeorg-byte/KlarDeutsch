# Умное заполнение форм слов через AI

## Проблема
В базе данных есть слова без заполненных грамматических форм:
- **Существительные** без `plural` (например, `die Unterschrift` без `die Unterschriften`)
- **Глаголы** без `verb_forms` (например, `gehen` без `gehen, ging, ist gegangen`)
- **Прилагательные** без форм (им они не нужны): `groß`, `neu`

Раньше скрипты пытались угадать тип слова по эвристикам, что было неточным.

## Решение
Скрипт `smart_fill_missing_forms.py` использует AI для:
1. **Определения типа слова** (существительное/глагол/прилагательное/фраза)
2. **Заполнения соответствующих форм**:
   - Существительные: `article` + `plural` (или `—` если нет мн.ч.)
   - Глаголы: `verb_forms` в формате "Infinitiv, Präteritum, Partizip II"
   - Прилагательные: пропускает (им не нужны эти формы)
   - Фразы: пропускает

## Использование

### 1. Диагностика
```bash
python tools/check_missing_forms.py
```
Покажет статистику и примеры слов без форм.

### 2. Тестовый режим (без записи в БД)
```bash
python tools/smart_fill_missing_forms.py
```
Покажет, что AI определил для каждого слова, но **не запишет** в базу.

### 3. Рабочий режим (с записью в БД)

Открой `tools/smart_fill_missing_forms.py` и измени:
```python
DRY_RUN = False  # Было True
```

Затем запусти:
```bash
python tools/smart_fill_missing_forms.py
```

Скрипт обработает **50 слов** за один запуск (настройка `BATCH_SIZE`).

### 4. Повторная обработка

Для обработки следующей партии слов просто запусти скрипт снова:
```bash
python tools/smart_fill_missing_forms.py
```

Скрипт берёт слова, отсортированные по `ai_checked_at ASC NULLS FIRST`, то есть сначала непроверенные.

## Что записывается в БД

| Тип слова | Поле article | Поле plural | Поле verb_forms |
|-----------|--------------|-------------|-----------------|
| **Существительное** | ✅ der/die/das | ✅ форма или `—` | ❌ (пусто) |
| **Глагол** | ❌ (пусто) | ❌ (пусто) | ✅ формы |
| **Прилагательное** | ❌ (пусто) | ❌ (пусто) | ❌ (пусто) |
| **Фраза** | ❌ (пусто) | ❌ (пусто) | ❌ (пусто) |

## Маркер отсутствия множественного числа

Если существительное не имеет множественного числа, в поле `plural` записывается `—` (прочерк).

**В интерфейсе** это отображается как:
- `Pl: нет мн.ч.` (серым цветом)

Примеры слов без мн.ч.:
- `das Geld` (деньги)
- `die Milch` (молоко)
- `der Mut` (мужество)

## Обработка ошибок и лимитов API

Скрипт автоматически обрабатывает:

### 1. **Лимиты API (Rate Limiting)**
- При превышении лимита Groq API скрипт ждёт и повторяет запрос
- Время ожидания увеличивается с каждой попыткой (2с, 4с, 6с...)
- После исчерпания всех попыток слово пропускается

### 2. **Ошибки парсинга JSON**
- Если AI вернул невалидный JSON, ошибка логируется
- Повторных попыток не делается (бессмысленно)

### 3. **Сетевые ошибки**
- Таймауты, обрывы соединения и т.д.
- Автоматический повторный запрос (до MAX_RETRIES)

### Что делать при частых лимитах:

1. **Уменьшить BATCH_SIZE** (например, до 50)
2. **Увеличить RETRY_DELAY** (например, до 5 секунд)
3. **Сменить модель** на более быструю (`gemma2-9b-it`)
4. **Запускать в несколько этапов** - по 100 слов с перерывами

## Пошаговая инструкция

### Шаг 1: Проверь статистику
```bash
python tools/check_missing_forms.py
```

### Шаг 2: Тестовый запуск (без записи)
```bash
# Убедись, что DRY_RUN = True
python tools/smart_fill_missing_forms.py
```

Проверь вывод:
- ✅ Правильно ли определяется тип слов?
- ✅ Корректные ли формы предлагаются?

### Шаг 3: Рабочий запуск
```bash
# Измени в скрипте: DRY_RUN = False
python tools/smart_fill_missing_forms.py
```

### Шаг 4: Повторяй пока не обработаешь все слова
```bash
# Просто запускай скрипт снова и снова
python tools/smart_fill_missing_forms.py
python tools/smart_fill_missing_forms.py
python tools/smart_fill_missing_forms.py
```

Каждый запуск берёт следующую партию из очереди (слова без `ai_checked_at`).

## Интеграция с основной AI проверкой

После заполнения форм через скрипт, перезапусти AI проверку из админки:

1. Зайди в админку → Дашборд
2. Нажми "Сбросить" (очистить статус проверки)
3. Нажми "AI проверка слов"

Основной AI check endpoint (`/api/admin/check-words`) теперь тоже:
- ✅ Определяет тип слова
- ✅ Заполняет `plural` для существительных (или `—` если нет мн.ч.)
- ✅ Заполняет `verb_forms` для глаголов
- ✅ Пропускает прилагательные

**Рекомендуемый порядок действий:**
1. Сначала запусти `smart_fill_missing_forms.py` для массового заполнения
2. Затем запусти AI проверку из админки для обогащения (примеры, синонимы, и т.д.)

## Настройки

В начале скрипта `smart_fill_missing_forms.py`:

```python
BATCH_SIZE = 100     # Сколько слов обрабатывать за один запуск
DRY_RUN = False      # True = только просмотр, False = запись в БД
MAX_RETRIES = 3      # Максимум повторных попыток при ошибке API
RETRY_DELAY = 2      # Задержка между попытками (секунды)
MODEL_NAME = "qwen-3-32b"  # Используемая модель
```

### Рекомендуемые значения:

| Параметр | Тест | Быстрая обработка | Макс. скорость |
|----------|------|-------------------|----------------|
| BATCH_SIZE | 20 | 100 | 200 |
| MAX_RETRIES | 5 | 3 | 2 |
| MODEL_NAME | qwen-3-32b | qwen-3-32b | gemma2-9b-it |

### Доступные модели Groq:

- `qwen-3-32b` - **рекомендуется**, быстрый и точный
- `llama-3.3-70b-versatile` - более точный, но медленнее
- `mixtral-8x7b-32768` - хороший баланс
- `gemma2-9b-it` - самый быстрый, но менее точный
